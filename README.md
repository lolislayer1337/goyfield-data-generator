# goyfield data generator

Инструмент для выкачивания нужной информации из файлов игры.

## Данные и папки

* Файлы `.json`, которые вы вытащили, нужно поместить в папку `./data` (ее по умолчанию нет, надо создать)
* Изображения - в папку `./images`
    * `./images/items` - иконки предметов
    * `./images/buildingIcons` - иконки строений (те которые белые)
* Уже существующие данные, которые нужно объединить с новыми - в `./merge`
    * `./merge/data` - объединенные таблицы с инфой
    * `./merge/locales` - объединенные таблицы с локализацией
* Выходные данные функций находятся в `./out`
    * `./out/data` - выходные таблицы с инфой
    * `./out/locales` - выходные таблицы с локализацией

## Функции и файлы

### main.py

Главный файл, в нем находятся функции для запуска (включать/выключать комментированием).

### paths.py

Файл с путями до файлов.

### create_empty_merged_files_if_not_exists()

Создает пустые файлы в `./merge`, если они не существуют.

### fetch_data()

Достает необходимую информацию из `./data`, преобразует ее и сохраняет в `./out/data`.

### fetch_locales()

Достает локализацию из `./data`, данные из `./out/data`, делает таблицы с локализацией и сохраняет их в `./out/locales` для каждого языка.

### merge_data()

Достает таблицы из `./merge/data` и из `./out/data`, объединяет их и сохраняет в `./merge/data`. Если появляется конфликт, то данные из `./out/data` имеют приоритет.

### merge_locales()

Достает таблицы из `./merge/locales` и из `./out/locales`, объединяет их и сохраняет в `./merge/locales`. Если появляется конфликт, то данные из `./out/locales` имеют приоритет.

### check_images()

Проверяет что для всех необходимых предметов и строений в папках `./images/items` и `./images/buildingIcons` есть иконки. Если нет иконки с нужным iconId, его покажет в консоли. Проверка происходит по iconId и названию изображений.

### check_unused_images()

Проверяет папки `./images/items` и `./images/buildingIcons` на наличие лишних иконок и предлагает удалить их, если такие есть. Проверка происходит по iconId и названию изображений.
